DEPLOYMENT INSTRUCTIONS FOR VERCEL
===================================

STEP 1: Go to Vercel
- Visit https://vercel.com
- Sign in with your GitHub account

STEP 2: Import Project
- Click "Add New" then "Project"
- Click "Import Git Repository"
- Select "hariharanm-ds/Agentic-Honeypot"
- Click "Import"

STEP 3: Configure Project
- Framework: Flask (auto-detected)
- Root Directory: ./
- Environment Variables: (can skip, defaults already set)
- Click "Deploy"

STEP 4: Wait for Deployment
- Watch the build logs
- Deployment takes 3-5 minutes
- You'll see "Ready" when complete

STEP 5: Get Your Live URL
- Copy the URL from the dashboard
- Format: https://[project-name].vercel.app

STEP 6: Update Frontend (Important!)
- Open advanced_ui.html
- Find: const API_BASE = 'http://localhost:5000';
- Change to: const API_BASE = 'https://[your-vercel-url]';
- Save and commit: git add advanced_ui.html && git commit -m "Update API URL" && git push
- Vercel auto-redeploys from GitHub

THAT'S IT! Your app is live!

TROUBLESHOOTING
===============

If Vercel deployment fails:
1. Check build logs for errors
2. Verify wsgi.py exists in root directory
3. Verify requirements.txt has all dependencies
4. Try deleting and redeploying

If API connection fails:
1. Check the URL in advanced_ui.html is correct
2. Verify no typos
3. Check CORS is enabled (it is)
4. Verify X-API-Key header is being sent

If scam detection times out on Vercel:
- Vercel has 30-second limit
- Use Railway instead (no time limit)

API ENDPOINTS
=============

POST /api/v1/conversation
- Create new conversation

POST /api/v1/conversation/{id}/message
- Send message to conversation

GET /api/v1/conversation/{id}
- Get conversation details

GET /api/v1/conversation/{id}/export
- Export conversation

DELETE /api/v1/conversation/{id}
- Delete conversation

GET /health
- API health check

All endpoints require X-API-Key header (test_key_12345 for local testing)
